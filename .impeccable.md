## Design Context

### Users
Cerebro is used by Stevie as a local-first intelligence workbench for finding high-signal developer tools, repositories, people, and technical patterns. The user is often scanning quickly, comparing candidates, deciding whether a result deserves vault permanence, and generating repo or developer skills for later agent work.

### Brand Personality
Cerebro should feel calm, exacting, and premium. It is not a marketing site or playful dashboard; it is a serious operator console for personal research and agentic engineering. The interface should evoke confidence, focus, and a sense that the system is quietly doing deep work.

### Aesthetic Direction
Use a dark neomorphic workbench direction: rich depth, soft raised and inset surfaces, generous negative space, restrained glow, and strong information hierarchy. The palette should avoid one-note monochrome; combine graphite, deep green, blue-gray, amber, and controlled rose for state and emphasis. The first screen must remain the usable workbench, not a landing page.

### Design Principles
- Prioritize calm scanability: fewer cramped surfaces, more breathing room, clear columns, and stable controls.
- Make depth functional: use neomorphic shadow/inset treatments to distinguish search, results, details, and process state without decorative clutter.
- Preserve operator density where it matters: results, reasons, stage logs, and artifact paths must stay visible, readable, and fast to compare.
- Treat generated artifacts as first-class outputs: skill links, scan state, and vault paths need strong visual presence.
- Keep UI resilient: no text overlap, no layout shift during loading/generation, accessible focus states, and mobile-first fallbacks.
